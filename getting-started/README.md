# Getting Started — Thriving Families

## You have 48 hours. Start here.

### Step 1 (15 minutes): Orient
- Read `QUICKSTART.md` — it is short and tells you the most important constraints
- Skim `01_problem_space/02_targeted_problem_statements.md` — both statements
- Read `03_artifacts/youth_data_feasibility.md` if you are considering youth employment

### Step 2 (30 minutes): Choose
- Discuss as a team: which problem, which user, what can we actually build?
- Fill out `99_templates/working_direction_note.md`
- Confirm your choice with `99_templates/decision_memo.md`

### Step 3 (60–90 minutes): Research
- Run the most relevant prompts from `05_prompts/research/` using the Perplexity runner
- Or read the pre-seeded content in this repo as your starting research
- Log confirmed sources in `evidence_log.md`

### Step 4 (30 minutes): Scope
- Read `04_build_guides/01_mvp_shapes.md` — pick one shape
- Read `04_build_guides/02_recommended_architectures.md` — pick one stack
- Fill out `99_templates/project_one_pager_template.md`

### Step 5: Build
- Keep the demo path to 3–5 steps
- Prioritize content curation over feature completeness
- Test on mobile before Sunday

### Step 6 (before demo): Risk review
- Run through `05_prompts/06_risk_review_prompt.md`
- Read `04_build_guides/05_demo_advice.md`

## Critical reminders

- Youth Employment has D3=1. There is no dataset waiting for you. You will curate content from public websites. Plan for 2–3 hours of content work before you write code.
- Maternal Health requires data access verification. Get a Census API key before Friday if possible (https://api.census.gov/data/key_signup.html).
- Do not store personal information about minors. This is non-negotiable.
- Every claim needs a source. Every source needs a link.
- "Verify directly with the program" is your best friend as a disclaimer.
