# Risk Review Prompt

Use this prompt before demoing. Run it against your prototype copy, UI text, README claims, and demo script.

---

## Checklist — Youth Employment Tools

- [ ] Does any copy claim a program is definitely open or accepting applications? (Remove if yes — say "verify with the program")
- [ ] Does any copy state a teen is eligible for a specific program? (Remove eligibility determinations)
- [ ] Does the work permit guidance accurately reflect current Virginia DOLI requirements? (Verify against official source)
- [ ] Does the tool store any personal information about teens? (Must be no — remove if yes)
- [ ] Does any copy claim the tool is City-approved or officially endorsed? (Remove if yes)
- [ ] Does any feature request name, age, address, or other identifying information from a teen? (Remove or justify legally)
- [ ] Are all program listings linked to official sources? (Add links if missing)
- [ ] Does the tool claim to match a teen with an employer? (Clarify as discovery, not placement)

---

## Checklist — Maternal Health Data Tools

- [ ] Does any copy make a clinical recommendation to a patient? (Remove — this tool is for researchers)
- [ ] Is each indicator linked to a named source with an access date? (Add citations if missing)
- [ ] Does any statistic differ from the official source by more than rounding? (Verify and correct)
- [ ] Does the tool claim real-time data if data is actually cached or downloaded? (Clarify vintage date)
- [ ] Does any copy state a health organization endorses the dashboard? (Remove unless explicitly confirmed)
- [ ] Does the indicator dictionary define terms consistently with the named source? (Verify definitions)
- [ ] Is there a clear disclaimer that this tool supports expert analysis, not clinical decision-making? (Add if missing)

---

## Shared Checklist

- [ ] Does any copy use language like "ensures," "guarantees," or "will" without certainty? (Replace with "helps," "supports," "guides")
- [ ] Are all links to official sources working? (Test each one)
- [ ] Is the tool mobile-friendly? (Test on a phone)
- [ ] Does the demo path work without an internet connection? (Prepare a fallback)
- [ ] Does the README or about page explain what the tool does NOT do? (Add if missing)

---

## What to do if you find an issue

1. Remove or soften the claim
2. Add a source citation
3. Add a disclaimer
4. Remove the feature if it cannot be made safe in time

Do not demo anything that you cannot defend to a judge who asks "what happens if someone relies on this and it is wrong?"
